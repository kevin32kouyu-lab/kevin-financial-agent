# Security Policy

Financial Agent 会处理 API Key、研究历史、账户会话和导出报告，因此默认按“最小暴露”原则维护。

## 支持范围

当前主线分支和最新发布分支接受安全修复。旧实验分支、个人部署配置和本地数据文件不在支持范围内。

## 报告安全问题

如果你发现漏洞，请优先使用 GitHub Security Advisory 或私有方式联系维护者。不要在公开 issue 里粘贴真实密钥、会话 token、数据库文件或用户报告内容。

报告时请尽量包含：

- 受影响的版本或 commit
- 可复现步骤
- 影响范围
- 你已经确认没有公开的敏感信息

## 密钥与配置

- `.env`、Railway 变量、API Key、cookie secret 和数据库文件不能提交到 git。
- `.env.example` 只能保存占位符。
- `scripts/check_secrets.py` 会扫描明显的密钥格式，CI 会通过统一验证入口执行它。
- 如果密钥进入 git 历史，应立即轮换密钥，并把旧密钥视为已泄露。

## 会话与账户

- 本地账户会话依赖服务端 session secret。
- 生产环境必须设置强随机 secret，不要使用示例值。
- 管理员接口应只暴露给可信环境，并保留审计事件。

## 投资与数据安全边界

- 本项目输出是研究辅助信息，不是财务、法律或税务建议。
- 报告、PDF 和历史记录可能包含用户输入和偏好，不应公开上传。
- 历史数据、新闻、RAG 证据和回测结果都可能降级；系统应明确展示限制，而不是静默伪装完整数据。
