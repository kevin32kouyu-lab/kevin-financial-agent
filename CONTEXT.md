- 正在把“完整演示版 + 浏览器级长期记忆”整理成统一版本，并准备合并回 `main`。
- 当前统一版本所在分支是 `codex/unified-demo-memory`。
- 这条线会作为后续 GitHub 与 Railway 的正式演示基础版本。

## 上次停在哪个位置
- 已把完整演示版上的首页、终端、回测、历史页保留下来，没有再退回旧版本。
- 已把长期记忆升级为“按浏览器隔离”：前端自动生成 `client_id`，后端按 `X-Client-Id` 保存和读取偏好。
- 结果页已经加入“长期记忆”卡片，支持查看、编辑、保存、恢复和清空。
- 当前代码已验证通过：前端构建通过，后端测试通过。

## 近期关键决定与原因
- 不再把“完整演示版”和“长期记忆版”分成两条线，而是统一成一个版本，避免演示时功能割裂。
- 长期记忆不新开登录系统，先按浏览器隔离，是为了最快做出稳定、可演示、少重复追问的效果。
- 继续复用现有 `user_preferences` 存储，并把 `profile_id` 直接设为 `client_id`，是为了降低改动风险并保持数据库兼容。
- 长期记忆只记资金、币种、风险、期限、风格、偏好板块和偏好行业，不记 ticker 和一次性筛选条件，是为了避免把单次任务误记成长期偏好。
- Railway 持久化卷必须挂到 `/app/data/runtime`，不能挂到 `/app/data`，否则会覆盖镜像内的种子数据目录。
