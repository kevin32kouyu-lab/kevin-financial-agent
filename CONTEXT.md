## 近期关键决定与原因
- 简历主 bullet 不写“后端测试 125 项通过、前端构建通过”，这类内容更适合作为面试补充；主文案改用流程、角色、证据、校验、接口和产品闭环的量化表达。
- 简历项目经历应重点体现“完整可用的投研 Agent 产品”，突出多角色流程、RAG 证据、三报告、回测、PDF 导出和部署验证，避免只写成普通数据抓取或聊天机器人。
- `legacy/` 暂时保留：当前 `app/analysis_runtime/live_data.py` 和部分测试仍引用其中的数据函数，直接删除会影响运行。
- 清理只处理旧备份、测试产物、临时目录和缓存，不处理运行数据库、真实环境变量、本地依赖和项目 worktree，避免影响本地运行。
- 账户系统先做本地邮箱密码，不接第三方 OAuth：实现快，适合单服务 Railway 部署。
- 税费只做简化 flat-rate，不做具体地区税务建议。
- 分红只在数据里有现金分红列时纳入，否则明确写入限制说明。
- 自动刷新任务默认关闭，用环境变量开启，避免部署资源不可控。
- `.npm-cache/` 仍是原本存在的未跟踪目录，本轮未处理。
- PDF 导出采用后端统一模板：保留用户原始问题、结论摘要、评分表、逐票卡、风险、完整 memo 和最近回测摘要，避免前端临时 HTML 与页面内容不一致。
- 面试开源项目优先：文档应聚焦产品能力、架构、运行方式和测试结果，不保留临时展示入口。
- Tool Registry 先落地为内部基础层：先统一权限、超时、重试、缓存和审计结果结构，再逐步把现有数据链路接入。
- 三报告来自同一次 run：`simple_investment` 面向普通用户快速决策，`professional_investment` 面向专业研究深度，`development` 面向工程可信度说明。
- `report_outputs.investment` 暂时保留为兼容别名，指向 `simple_investment`；`final_report` 继续指向简单版正文。
- PDF 导出扩展到 `kind=simple_investment/professional_investment/development`；旧 `kind=investment` 映射到简单版。
- 开发报告 diagnostics 统一以 `evidence_count`、`validation_check_count` 为主字段，同时保留旧别名一版兼容历史 run。
- 投资报告图表合同固定为“三类条形图 + 一类回测折线图”：风险图必须带数值，回测图固定使用 `points + summary`。
- 结论页不自动创建回测，但如果当前已经加载过回测，前端会临时覆盖回测图，保证页面、导出和回测页尽量一致。
- 终端进度条改为“阶段加权”而不是“完成步数粗算”：因为 run 只在阶段结束后才写入 step，直接按数量算会在报告生成阶段假卡住。
- EventSource 不能自定义请求头，所以匿名用户的事件流不能再只依赖 `X-Client-Id`：前端改为把 `client_id` 放进事件流 URL，后端同时支持从 query string 读取。
- 投资 PDF 默认采用“平衡版”而不是完整长 memo：PDF 负责可读、可分享，详细长内容继续留在网页、HTML 和 Markdown 导出里。
- PDF 图表统一走“有效图表”逻辑：如果当前 run 已有回测，就优先放组合 vs 基准图；没有回测时，第三张图自动切换成风险贡献图。
- 首页继续保留品牌感和动态背景，但文案必须直接面向用户，不再像项目介绍页。
- 开始研究页采用“三种起点 + 自然语言输入 + 结构化补充 chips + 系统四步预览”组合：用户先判断自己是单次提问、组合研究还是持续跟踪，再进入提问。
- 账户系统沿用现有本地邮箱密码接口，但前台必须给出清晰入口和“同步浏览器记忆”动作，才能让跨设备记忆对用户真正可见。
- 历史页除了回看，还要支持“一键继续跟进”：把旧问题带回提问页，默认转成基于最新数据的连续研究。
- 持续跟踪先做轻量本地列表，不新增复杂后端订阅系统，先验证真实使用感受。
- 用户引导只放在普通终端前台，不进入 `/debug`：普通用户看到功能说明，开发细节继续留在调试页。
- 终端交互优化优先保证“先切页面、再加载数据”：重任务可以继续异步等待，但按钮、页签和报告标签必须先给即时反馈。
- 运行进度采用“真实阶段 + 前端连续显示进度”：阶段文案仍来自后端步骤，百分比在任务运行时持续推进到 98%，完成后归 100%，避免用户误以为卡死。
- 运行中兜底轮询只依赖当前 run 和状态，不再依赖会随重渲染变化的函数引用，避免进度动画把轮询计时器不断清掉。
- 模型供应商采用单一路由：只使用 DeepSeek 的 OpenAI 兼容接口，避免不同供应商生成口径不一致；真实 Key 只能放在本地 `.env` 或部署平台环境变量，不能写入仓库。
- 项目内手动 worktree 固定放到 `.worktrees/`：目录在 E 盘项目内，并已加入 `.gitignore`，避免误提交工作区内容。
- 新手引导采用前端静态示例报告：让新用户先看到完整结论和回测，不消耗模型额度，也不污染真实历史记录。
- 前端不再通过 CSS `@import` 加载外部 Google Fonts：避免首页跳终端时被外部字体网络请求卡住首屏渲染。
- 示例引导里的历史页只做说明，不读取后端历史列表：避免 `demo-guide-run` 和真实历史刷新混在一起，也减少最后一步的视觉抖动。
