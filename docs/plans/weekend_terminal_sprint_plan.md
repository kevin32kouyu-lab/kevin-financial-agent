## 背景与目标

这份计划用于本周末快速产出一个“可展示、可讲解、可稳定演示”的 Terminal 版本。  
本轮只聚焦展示端体验，不做大规模底层重构。  

本轮锁定策略：

- 只做 Terminal 展示体验
- `/debug` 不在前台导航展示，但后端能力继续保留
- 保持现有接口兼容，避免演示前回退

## 每阶段交付物

- 一版可展示的 Terminal 页面（更友好、更易懂）
- 一套可执行的研究流程（实时 + 历史 + 回测）
- 一份可导出的正式报告（中英一致）
- 一份周末版本说明（做了什么、可演示什么）

## 验收标准

1. 用户可在 Terminal 完成：输入需求 -> 生成报告 -> 查看回测 -> 导出报告。  
2. 页面具备引导、反馈、提示，非技术用户也能理解。  
3. 连续运行时，不出现“无提示失败”或“空白区域”。  
4. 中文输入输出中文，英文输入输出英文，不混杂。  
5. `/debug` 不出现在前台导航中。  

## 风险与应对

- 风险：免费数据源限流，可能导致部分模块短时失败。  
  应对：保持主源 + 备用源 + 缓存，并输出可读原因。  

- 风险：周末需求集中，交互改动过多可能影响稳定性。  
  应对：先稳主流程，再做视觉细节，保证核心路径优先。  

## 验证问题（周末版）

### Query 2（英文，成长风格）

I have $100,000 and want long-term growth with controlled risk. Please compare MSFT, GOOGL, AMZN and META, then give me a formal investment memo with staged entry advice.

### Query 3（历史回测）

请以 2025-10-01 作为历史研究时点，给出同样风格的投资建议，并回放到 2026-04-13，展示组合收益与 SPY 对比。
